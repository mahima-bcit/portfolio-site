Portfolio CMS

2026

A solo full-stack CMS for managing a developer portfolio. Three-tier role-based access control, image upload pipeline with safe-delete guards, and security audit logging across the entire request lifecycle.

Node.js Express MongoDB EJS

The challenge

Build a full-featured CMS from scratch as a solo project: public browsing for portfolio visitors, authenticated admin CRUD for content management, and the kind of role hierarchy and audit trail you would expect in a real production application. No frameworks beyond Express; the architecture is mine end to end.

My role

Solo developer. Designed the data model, architected the role-based access control system, implemented the image upload pipeline with referential integrity guards, and built the audit logging layer that captures every access denial.

What I built

  • 25+ Express routes covering public browsing and admin CRUD for projects, skills, education, and experience
  • Four Mongoose models with relationships and validation
  • Three-tier role-based access control (USER, MODERATOR, ADMIN) with custom middleware
  • bcrypt password hashing and MongoDB session storage via connect-mongo
  • Multer-based image upload pipeline with safe-delete guards preventing deletion of referenced data
  • Security audit logging capturing timestamp, user ID, role, HTTP method, path, and IP on every access denial

Key takeaways

Solo full-stack work forced me to think about the whole system at once. The audit logging design in particular started as a small idea and grew into one of the cleanest parts of the codebase, because every access decision had to flow through one well-defined chokepoint. That clarity carried over into how I think about middleware in any Express app now.
